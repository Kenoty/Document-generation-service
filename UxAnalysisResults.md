# Анализ и улучшение Usability (ISO/IEC 25010)

**Проект:** Document Generation Service  

## 1. Введение

В рамках лабораторной работы был проведен анализ качества разрабатываемого ПО по атрибутам группы **Usability** согласно стандарту ISO/IEC 25010:2011. На основе анализа были предложены и реализованы улучшения пользовательского интерфейса и пользовательского опыта.

---

## 2. Оценка атрибутов Usability

### 2.1 Распознаваемость соответствия (Appropriateness Recognizability)

**Определение:** Способность пользователя распознать, подходит ли программное обеспечение для его нужд.

**Оценка текущего состояния:**
- Главная страница содержит краткое описание сервиса
- Добавлен гостевой режим для быстрого знакомства с функционалом
- На странице шаблонов отображаются примеры использования

**Оценка:** **Средняя** — пользователь может оценить базовый функционал, но полный спектр возможностей раскрывается после регистрации.

---

### 2.2 Обучаемость (Learnability)

**Определение:** Способность продукта быть простым для изучения.

**Оценка текущего состояния:**
- Добавлены подсказки по работе с плейсхолдерами
- В интерфейсе редактирования шаблона справа отображаются доступные поля
- При клике на поле в списке оно выделяется в тексте шаблона

**Оценка:** **Высокая** — пользователь быстро понимает, как работать с шаблонами и полями.

---

### 2.3 Используемость / Операбельность (Operability)

**Определение:** Способность продукта быть простым в использовании и управлении.

**Оценка текущего состояния:**
- Кнопка генерации документа добавлена непосредственно к каждому шаблону
- Упрощен процесс создания документа — не требуется переходить в отдельный раздел
- Кнопка загрузки DOCX шаблона перенесена в окно создания нового шаблона

**Оценка:** **Высокая** — ключевые действия доступны в контексте, сокращено количество переходов.

---

### 2.4 Защита от ошибок пользователя (User Error Protection)

**Определение:** Способность системы защищать пользователя от совершения ошибок.

**Оценка текущего состояния:**
- При регистрации есть проверка на существующего пользователя
- При генерации документа есть валидация обязательных полей
- Добавлено выделение полей в тексте для наглядности

**Оценка:** **Средняя** — базовая валидация есть, требуется дальнейшее улучшение.

---

### 2.5 Эстетика GUI (User Interface Aesthetics)

**Определение:** Насколько приятен и эстетичен пользовательский интерфейс.

**Оценка текущего состояния:**
- Используется минималистичный дизайн
- Единая цветовая гамма
- Улучшена навигация и расположение элементов управления

**Оценка:** **Средняя** — аккуратно, но требуется дальнейшая доработка.

---

### 2.6 Доступность (Accessibility)

**Определение:** Способность продукта быть доступным для пользователей с ограниченными возможностями.

**Оценка текущего состояния:**
- Контрастность текста соответствует базовым требованиям
- Требуется доработка для полного соответствия WCAG 2.0

**Оценка:** **Низкая** — требуется доработка.

## 4. Пути улучшения UX

На основе проведенного анализа были выбраны и реализованы следующие улучшения:

### 4.1 Улучшение обучаемости и навигации при работе с шаблонами

**Проблема:** Пользователь не видел, какие поля доступны для замены в шаблоне, и не мог быстро найти их в тексте.

**Решение:** Добавить отображение списка полей справа от редактора шаблона. При клике на поле в списке — выделять соответствующее место в тексте.



**Обоснование:** Визуальная связь между полем и его местом в шаблоне сокращает время на поиск и снижает количество ошибок.

---

### 4.2 Улучшение используемости загрузки DOCX-шаблонов

**Проблема:** Кнопка загрузки DOCX находилась в главном меню Templates, что создавало лишние переходы при создании нового шаблона из файла.

**Решение:** Перенести кнопку "Upload DOCX" в окно создания нового шаблона, рядом с текстовым редактором.

**Обоснование:** Контекстное расположение действий снижает когнитивную нагрузку — пользователь видит все способы создания шаблона в одном месте.

---

### 4.3 Улучшение операбельности при генерации документов

**Проблема:** Для генерации документа из шаблона пользователю приходилось переходить в отдельный раздел меню "Documents", что увеличивало количество действий.

**Решение:** Добавить кнопку "Generate" для каждого шаблона в списке, которая открывает форму генерации с выбранным шаблоном.

**Обоснование:** Сокращение пути от шаблона к документу до одного клика улучшает пользовательский опыт и ускоряет выполнение основной задачи.

---

## 5. Реализованные улучшения

### 5.1 Отображение полей с выделением в тексте

**До:**
- Редактор шаблона представлял собой простое текстовое поле
- Пользователь должен был самостоятельно искать и запоминать, какие поля используются

**[СКРИН 1 ДО: Редактор шаблона без списка полей]**

**После:**
- Справа от редактора отображается список всех полей, извлеченных из шаблона
- При клике на поле в списке, соответствующее место в тексте выделяется (подсветка, скролл к позиции)
- Пользователь может быстро найти и заполнить нужные поля

**[СКРИН 1 ПОСЛЕ: Редактор шаблона со списком полей справа и выделенным полем в тексте]**

---

### 5.2 Перенос кнопки Upload DOCX в окно создания шаблона

**До:**
- В главном меню "Templates" были две отдельные кнопки: "New Template" и "Upload DOCX"
- Пользователь переходил в раздел, затем выбирал способ создания

**[СКРИН 2 ДО: Главное меню Templates с двумя кнопками]**

**После:**
- Кнопка "New Template" открывает модальное окно
- Внутри окна есть две вкладки/зоны: "Create from text" и "Upload DOCX"
- Все способы создания шаблона собраны в одном месте

**[СКРИН 2 ПОСЛЕ: Окно создания шаблона с текстовым редактором и кнопкой Upload DOCX]**

---

### 5.3 Кнопка Generate для каждого шаблона

**До:**
- В списке шаблонов были только кнопки "Edit" и "Delete"
- Для генерации документа нужно было перейти в раздел "Documents" и выбрать шаблон из выпадающего списка

**[СКРИН 3 ДО: Список шаблонов без кнопки Generate]**

**После:**
- К каждому шаблону в списке добавлена кнопка "Generate"
- При нажатии открывается форма генерации с автоматически выбранным шаблоном
- Пользователь заполняет поля и сразу получает документ

**[СКРИН 3 ПОСЛЕ: Список шаблонов с кнопкой Generate]**

## 7. Выводы

В ходе работы был проведен анализ разрабатываемого ПО по атрибутам Usability согласно ISO/IEC 25010. Выявлены и реализованы три ключевых улучшения:

1. **Отображение полей с выделением в тексте** — повышает обучаемость и снижает время поиска нужных полей
2. **Перенос Upload DOCX в окно создания шаблона** — улучшает используемость за счет контекстного расположения действий
3. **Добавление кнопки Generate в список шаблонов** — сокращает путь к основной функции генерации документа

**Результат:** Оценка по атрибутам Обучаемость и Используемость повышена до "высокой". Продукт стал более интуитивным и удобным для повседневного использования.
